# Canva Image Map

Use this file as the production art checklist. The site includes SVG placeholders with these exact ratios. Replace the SVGs with Canva exports when ready.

## Export guidance

- Use `.webp` or optimized `.jpg` for photographic/collage imagery.
- Use `.png` only when transparency is needed.
- Keep file names the same if you do not want to edit JSON/HTML.
- Try to keep each image under 400 KB for fast loading.
- For product mockups, make the product the focus. Avoid tiny text and over-busy cover art.

## Specific creative direction

### Homepage hero collage — 1600 × 1000

Make this look like a curated indie convention table:

- 2–3 folded shirts
- A hoodie sleeve or product tag
- Sticker sheet
- Comic cover peeking out
- Print/poster corner
- Shipping label or packing slip
- Bold comic-book color accents
- No real copyrighted IP unless you own it

This is the most important image for avoiding the generic “AI-built GitHub site” look.

### Product mockups — 1200 × 1200

Each product image should feel like a real merch listing.

Recommended compositions:

- Apparel: shirt/hoodie centered with bold art placed on garment
- Prints: poster leaning on a table or taped to wall
- Stickers: scattered sticker sheet or kiss-cut layout
- Bundles: several items arranged like a convention table photo

### Title banners — 1600 × 600

Think of these as the title’s storefront marquee.

Include:

- Title logo
- Color palette
- Texture or setting cue
- Character silhouette or emblem if available
- Enough negative space to not fight with page text

### Title covers — 1000 × 1500

These should read as book identity cards, not necessarily final comic covers.

Include:

- Title name
- Genre mood
- Creator/publisher mark
- Strong central visual

### Creator avatars/logos — 800 × 800

Use:

- Publisher logo
- Creator mark
- Stylized portrait
- Monogram
- Imprint badge

Avoid low-res social profile screenshots.
